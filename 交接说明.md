# Kaduosi Atelier 高端品牌询盘站交接说明

## 1. 本次交付内容

本目录是基于 Alibaba 公开店铺资料重构的 **高端汽车内饰品牌站 + 询盘转化站**。

- `index.html`：自包含单文件页面，已内嵌产品数据，可直接双击打开。
- `data.json`：原始结构化数据备份，包含品牌信息、产品、买家画像和 SOP。
- `hero/`、`logos/`、`uploads/`、`textures/`：后续部署时用于自托管图片素材的目录。

## 2. 当前品牌定位

品牌名暂定：**Kaduosi Atelier**

品牌方向：**Premium Automotive Interior Programs**

核心表达：

> Build a cabin upgrade line buyers can feel.

这版不再把 Kaduosi 表达成普通汽车座套供应商，而是包装成：

> 能帮助汽配经销商、汽车改装店和私标买家建立高端汽车内饰升级产品线的品牌型供应商。

## 3. 高端品牌站结构

当前页面已从普通产品目录改为品牌叙事结构：

1. **豪华首屏 Hero**：用大字号品牌主张、香槟黑金视觉和 RFQ 表单建立第一印象。
2. **Atelier 品牌叙事**：从 supplier 转为 interior upgrade atelier。
3. **三大 Signature Programs**：
   - Luxury Custom Fit Series
   - Universal Dealer Series
   - Interior Protection Add-ons
4. **Material & Fit System**：材料方向、版型开发、私标包装、组合销售策略。
5. **Selected SKUs**：SKU 退居二级，用来证明产品能力，而不是让页面像平台货架。
6. **Commercial Proof**：公司、类目、买家类型等基础信任结构。
7. **Internal SOP**：`?staff` 或 `?admin` 可见，给业务员跟进使用。

## 4. 询盘转化结构

高端视觉下仍保留询盘转化闭环：

1. 首屏 **Private-label RFQ** 表单。
2. `Build my seat-cover program` 强 CTA。
3. `Request material sample plan` 样品路径。
4. 三大产品 Program 的 `Quote this program` 按钮。
5. 每个 SKU 的 `Quote this SKU` 按钮。
6. 底部固定 Sticky RFQ CTA。
7. `Sample-to-bulk workflow` 转化区。
8. 表单提交后自动生成结构化邮件草稿。

注意：当前销售邮箱尚未填写，邮件草稿会打开本机邮箱客户端，但收件人为空。正式上线前必须补充真实销售邮箱。

## 5. 页面入口

可以直接双击：

```text
index.html
```

如果需要模拟正式部署环境，推荐在本目录运行：

```bash
python3 -m http.server 8080
```

然后访问：

```text
http://localhost:8080/
```

三种视图：

| 入口 | 用途 |
|---|---|
| `/` | 买家公开视图 |
| `/?staff` | 业务员内部视图，可看到 Internal SOP |
| `/?admin` | 管理员视图标记，目前仍是本地原型 |

多语言入口：

| 入口 | 语言 |
|---|---|
| `/?lang=en` | English |
| `/?lang=zh` | 中文 |
| `/?lang=de` | Deutsch |
| `/?lang=es` | Español |

页面右上角也有语言下拉选择，切换结果会写入浏览器 `localStorage`。

## 7. 正式投放前必须补充

为了成为真正可投放的高端品牌站，建议补充：

1. 公司销售邮箱
2. WhatsApp / 电话
3. 正式 Logo 或品牌字标
4. 3-5 张高清工厂 / 办公室 / 仓库 / 材料图
5. 3-5 张真实车型安装效果图，最好横图
6. 证书或第三方验厂截图
7. 包装图、样品盒图、材料细节图
8. 目标市场优先级：北美、欧洲、中东、东南亚等

## 8. 后续部署说明

如需上线为公开品牌站，建议部署到 GitHub Pages。

部署后应交付：

1. 买家公开 URL
2. 业务员 URL：`?staff`
3. 管理员 URL：`?admin`
4. 管理员 token，需单独安全发送
5. 二维码：买家公开页 + 管理员页

## 9. 注意事项

- 公开页页脚不放 Alibaba 店铺链接，避免独立品牌站显得像平台橱窗。
- 当前图片仍为 Alibaba CDN 链接，正式部署前建议下载并自托管到 `hero/` 或 `uploads/`。
- 当前销售邮箱为空，必须补齐后再投放。
- 当前管理员功能为本地原型，部署 GitHub 后可接入 GitHub Contents API 作为轻量 CMS。
